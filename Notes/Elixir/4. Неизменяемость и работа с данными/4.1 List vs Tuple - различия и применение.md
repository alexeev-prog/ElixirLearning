## Определения и основные свойства

**Списки (List)** и **кортежи (Tuple)** — два фундаментальных типа коллекций данных в языке программирования Elixir. Оба типа являются неизменяемыми структурами, что означает невозможность изменения их содержимого после создания. Однако списки и кортежи имеют различные внутренние представления, свойства и области применения.
### Списки (List)

- Списки представляют собой структуры, упорядочивающие элементы в линейную последовательность.
- Обозначаются квадратными скобками: `[a, b, c]`.
- Списки могут содержать элементы любого типа, включая другие списки и кортежи.
- Элементы могут повторяться, а длина списка может быть произвольной.
- Списки реализованы на основе односвязного списка.

### Кортежи (Tuple)

- Кортежи представляют собой фиксированную коллекцию элементов, которая не изменяется по длине после создания.
- Обозначаются фигурными скобками: `{a, b, c}`.
- Элементы кортежа могут иметь различные типы, включая списки и другие кортежи.
- Длина кортежа фиксируется в момент его создания.
- Кортежи реализованы как непрерывный блок памяти.

## Внутренняя реализация и производительность

### Списки

- Каждый элемент списка хранит ссылку на следующий элемент (голова-хвост, head-tail).
- Добавление элемента в начало списка (`[new_element | list]`) выполняется за константное время `O(1)`.
- Доступ к элементу по индексу требует последовательного обхода от начала списка, что имеет линейную сложность `O(n)`.
- Списки эффективны для операций последовательного перебора и сопоставления с образцом (pattern matching).

### Кортежи

- Все элементы кортежа хранятся последовательно в памяти, что обеспечивает быстрый доступ по индексу `O(1)`.
- Изменение кортежа требует создания нового кортежа с копированием всех элементов, что имеет стоимость `O(n)`, где `n` — размер кортежа.
- Кортежи оптимальны для хранения фиксированных наборов данных, к которым требуется быстрый доступ по позиции.

## Неизменяемость и операции над структурами

- В Elixir все структуры данных, включая списки и кортежи, являются неизменяемыми (immutable). Любая операция, изменяющая структуру, создает новую копию с соответствующими изменениями.
- Операции добавления, удаления, замены элементов работают по-разному для списков и кортежей в силу различий внутреннего устройства.

## Сопоставление с образцом (Pattern Matching)

- Списки широко используются в сопоставлении с образцом для рекурсивной обработки данных, например при разложении на голову и хвост: `[head | tail]`.
- Кортежи применяются для сопоставления по фиксированному количеству элементов, например `{:ok, value}` или `{:error, cause}`.

## Практические применения

### Списки

- Используются для хранения произвольных последовательностей данных, размер которых может изменяться или быть неизвестен заранее.
- Применяются для построения коллекций, итерирования, фильтрации, отображения, аккумуляции значений.
- Являются предпочтительным выбором при необходимости частых добавлений или удаления элементов с начала коллекции.
- Широко используются для представления строк как списков символов (charlists).

### Кортежи

- Используются для хранения фиксированных по размеру наборов данных, где каждое положение имеет специфическое семантическое значение.
- Применяются для возврата множественных значений из функций, особенно для кодирования состояний: `{:ok, result}`, `{:error, reason}`.
- Являются основой для создания структур данных с фиксированной схемой, например, координат: `{x, y}`.
- Используются для передачи параметров между процессами в сообщениях.

## Сравнительная таблица свойств

|Свойство|Список (List)|Кортеж (Tuple)|
|---|---|---|
|Синтаксис|`[a, b, c]`|`{a, b, c}`|
|Внутренняя структура|Односвязный список|Блок памяти с фиксированными ячейками|
|Доступ к элементу по индексу|Линейный `O(n)`|Константный `O(1)`|
|Добавление в начало|Константное `O(1)`|Создание нового кортежа `O(n)`|
|Изменение элемента|Создание нового списка|Создание нового кортежа|
|Размер|Динамический|Фиксированный|
|Идеальное применение|Массовые коллекции, нефиксированный размер|Фиксированные структуры, быстрый доступ по позиции|

## Выводы о применимости

- Списки рекомендуется использовать для обработки последовательностей неизвестной или изменяющейся длины, а также для операций, связанных с последовательным перебором элементов.
- Кортежи предпочтительны для хранения фиксированных наборов значений, где важен быстрый доступ по позиции и размер известен заранее.
- Правильный выбор между списком и кортежем влияет на производительность, читаемость кода и поддерживаемость программ на Elixir.
